# Broker Pattern

- Also known as intermediary pattern
- Iniserts a middleman called a broker between service users (clients), and service providers (servers)
- Client is disconnected from the servers and has no knowledge of them
	- Asks a broker over a service interface and the broker forwards it to the appropriate server
	- Server returns outcome of action to the broker that forwards the result to the client that initiated the request
